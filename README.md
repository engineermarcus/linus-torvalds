# Linus Torvalds: The Guy Who Built the Internet's Foundation (Twice)

---

## First, Linux (1991)

Linus was a **21-year-old Finnish college student** who was basically just annoyed. He wanted a free, open operating system to tinker with on his PC — the existing options were either too expensive or too restricted.

So he did what any completely normal person would do — **he built his own operating system from scratch.**

He announced it on a mailing list in 1991 with possibly the most understated message in tech history, saying it was just a "hobby project" and "won't be big and professional."

That hobby project now runs **~96% of the world's servers**, every Android phone, and most of the internet's infrastructure. Just a small hobby.

Linux became the poster child of open source — thousands of developers worldwide contributing code for free. Which leads us to the problem...

---

## The Git Origin Story (2005)

By the early 2000s, the Linux project had thousands of contributors sending in code changes. They were using a version control tool called **BitKeeper** to manage it all.

Then in 2005, the relationship between the Linux community and BitKeeper's company broke down over a licensing dispute. BitKeeper said "you can't use our tool for free anymore."

Linus's response was very on-brand for him — instead of finding another tool, **he spent two weeks and built his own.**

Not a small tool. Not a basic tool. He built **Git** — one of the most influential pieces of software ever created — in roughly **14 days.**

He had a few goals:

- It had to be **fast** (Linux had thousands of files and contributors)
- It had to be **distributed** (no single point of failure)
- It had to be **reliable** (data corruption was not an option)
- And it had to be the **opposite of whatever BitKeeper was**

---

## What "Distributed" Actually Meant

Before Git, most version control systems had one central server. Everyone connected to it. If it went down, everyone stopped working.

Linus said no — **every developer should have the full history of the entire project on their own machine.** No single server owns the truth. Everyone has a complete copy.

This was a radical idea at the time and it's exactly why Git took over the world.

---

## The Name "Git"

Linus named it "Git" — which is British slang for **a stupid, unpleasant person.**

His exact quote was:

> *"I'm an egotistical bastard, and I name all my projects after myself. First Linux, now Git."*

The man has a sense of humor. A dark one, but still.

---

## Legacy

Linus handed off day-to-day Git maintenance pretty quickly to **Junio Hamano**, who still maintains it today. Linus went back to focusing on Linux.

But the impact was enormous:

- **GitHub** was built on top of Git in 2008
- Git is now used by **virtually every software company on earth**
- It completely replaced every competing version control system
